이미지 태그 img

닫는 태그가 없는 빈 태그
문서에 이미지를 삽입한다.
src:이미지 경로를 지정한다.
alt:이미지의 대체 텍스트를 입력한다.(웹 접근성, 시각장애인, 이미지 로딩 오류시)
width/height : 이미지의 크기를 지정한다, 한 가지만 정의해도 원본사이즈의 비율을 맞춰 변동된다.

상대경로와 절대경로

src 속성에는 이미지의 경로가 들어가며, 이미지의 상대경로와 절대경로가 있습니다.
상대경로는 현재 웹 페이지를 기준으로 상대적으로 이미지의 위치를 나타내는 경로이고,
절대경로는 실제 그 이미지가 위치한 곳의 전체 경로입니다 상대경로에서 ./는 페이지가 있는 현재 폴더를 나타냅니다.

이미지 파일 확장자 GIF vs JPG vs PNG

  • gif : 256색으로 제한적이지만 용량이 작고, 애니메이션과 투명이미지가 가능하다.
  • jpg : 높은 압축률과 자연스러운 색상 표현이 가능하며 사진이나 일반적인 그림에 사용한다. 투명이미지를 지원하지 않는다.
  • png : jpg와 비교했을 때, 이미지 손실이 없고 투명과 반투명 모두 지원한다.

폼 관련 요소

서버에 데이터를 전달하기 위한 요소들

  • input 요소 : 대표적인 폼 요소로, 다양한 type 속성으로 여러 종류의 입력 양식으로 나타난다.
  • type으로는 text, password, radio, checkbox, file, image, submit, reset, button...
  • 레이블 요소는 웹 접근성과 관련이 있다
  • fieldset : 여러 개의 폼 요소를 그룹화하여 구조적으로 만들기 위해 사용
  • legend : 폼 요소의 제목으로 fieldset 요소 내부에 작성
  • form 요소
  • action : 폼데이터를 처리하기 위한 서버의 주소
  • method : 데이터를 전송하는 방식을 지정(get-민감한 정보 전달시 사용하면 X, post)
